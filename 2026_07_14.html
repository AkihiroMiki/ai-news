OpenAI が GPT-5.6 家族（Sol / Terra / Luna）と ChatGPT Work を公開 ― 3 層価格体系と「ultra」マルチエージェント設定

Jul 9, 2026 新モデル

OpenAI が GPT-5.6 を 3 つのティアで同日公開。Sol はフラッグシップで大規模コードベースや長時間エージェント向け（$5/$30 per MTok）、Terra はオールラウンドのデフォルト（$2.50/$15）、Luna は小タスク向けの最安値モデル（$1/$6）。同時に ChatGPT Work エージェントを発表。複数エージェントを並列ストリームで協調させる「ultra」設定を搭載し、単なる回答生成でなく「仕事を丸ごと引き受ける」設計を謳っている。プロンプトキャッシュは明示的なブレークポイント指定と 30 分の最低キャッシュ保持に対応し、予測可能性が向上した。パブリックロールアウトは 6 月 26 日に政府承認の 20 組織向け限定プレビューを経て、7 月 9 日に一般公開へ移行した。

エンジニア的注目点: Terra の $2.50/$15 は Sonnet 5 の導入価格（$2/$10）とほぼ同価格帯で、二択の比較評価が現実的になった。ultra 設定は Agent SDK でどう叩けるかが実装者の関心事。キャッシュブレークポイントは長文プロンプトのコスト最適化に直結するため、プロンプト設計の見直し余地がある。

Meta が Muse Spark 1.1 と Meta Model API を公開 ― 初の有料 API、1M トークンコンテキストの自律エージェントモデル

Jul 9, 2026 新モデル

Meta Superintelligence Labs が Muse Spark 1.1 を発表し、同時に Meta Model API のパブリックプレビューを開始した。Meta が有料 API を提供するのはこれが初めてで、Llama のオープンウェイト路線とは異なるクローズドモデルとして位置付けられる。コンテキスト長は 1M トークンで、長時間タスク中の能動的なコンテキスト管理（重要ステップを保持しながら自律的に圧縮）が特徴。マルチエージェントオーケストレーターとしてもサブエージェントとしても機能し、コンピューター操作・ツール使用・コーディングで大きく向上している。価格は $1.25/$4.25 per MTok、新規 $20 クレジット付き。GPT-5.5 および Opus 4.8 とのエージェント評価で競合すると主張。

エンジニア的注目点: Meta が有料 API に踏み込んだことで OpenAI・Anthropic・xAI との直接競争に参入した。1M トークンの自律コンテキスト管理は長時間バッチ処理の実装コストを下げる可能性があり、$1.25/$4.25 という価格は同等性能のモデルと比較して競争力がある。ただしベンチマークは自社評価のみなので、独立検証を待ってから本採用判断をする方が安全。

Friendly Fire と GhostApproval ― AI コーディングエージェントの信頼境界を破る 2 つの PoC が同日公開

Jul 8–9, 2026 セキュリティ

7 月 8〜9 日、AI コーディングエージェントを狙う概念実証（PoC）が 2 件同時期に公開された。Friendly Fire（AI Now Institute）は、README などの通常ファイルに潜ませたプロンプトインジェクションにより、Claude Code（auto モード）と OpenAI Codex に悪意あるバイナリを実行させることに成功。信頼できないサードパーティ製コードの「セキュリティレビュー」という正当な作業を起点に攻撃が成立する。GhostApproval（Wiz Research）はシンボリックリンクの未チェックを利用した攻撃で、ユーザーには無害なファイルへの書き込み確認を表示しつつ、実際にはシステムの重要ファイルに書き込む。Claude Code では「エージェントの内部推論は正しい書き込み先を把握していたが、ユーザーへの確認プロンプトは別のファイル名を表示していた」という乖離が報告されている。Claude Code・Amazon Q Developer・Cursor・Augment・Windsurf で動作確認済み。

エンジニア的注目点: 両攻撃に共通するのは「信頼境界の設計ミス」で、エージェントが自律実行する範囲を最小化していない限り、承認プロンプトは実質的なセキュリティ防御にならない。外部リポジトリを触らせるワークフローには即時見直しが必要。auto モードで未知のコードをレビューさせる運用は当面停止が推奨される。

Claude Code がデフォルト権限モードを Manual に変更 ― CLI・VS Code・JetBrains 拡張を含む全プラットフォームで適用

Jul 2026 セキュリティ

Anthropic が Claude Code のアップデートを配布し、デフォルトの権限モードを Manual（全操作に都度確認が必要）に変更した。適用範囲は CLI・help 出力・VS Code 拡張・JetBrains プラグインの全プラットフォーム。従来はより広い権限を持つモードがデフォルトだったが、今回の変更によりエージェントによるファイル変更・コマンド実行・ネットワークアクセスにユーザーの明示的な承認が求められるようになる。Friendly Fire および GhostApproval の PoC 公開と同週に実施されたアップデートで、自律実行の安全設計を見直す動きとして注目される。

エンジニア的注目点: 既存の自動化スクリプトや CI パイプラインで Claude Code を auto モードで呼んでいる場合は、Manual モードへの移行で確認フローが挟まるようになり、動作変更が生じる。--permission-mode bypass 等のフラグで明示的に自動実行を有効化する必要があるが、それ自体が攻撃対象になりうる点を踏まえてアーキテクチャを見直す好機。

Fable 5 が全プラン有料化 ― 7 月 8 日から全サブスクライバーに利用クレジットが必須に

Jul 8, 2026 開発者向け

7 月 8 日より、Anthropic の最上位モデル Claude Fable 5（旧称 Claude 3.7 Sonnet 系の後継）へのアクセスに全サブスクライバー層で利用クレジットが必須となった。これまで Pro プランなど一部プランでは無制限アクセスが提供されていたが、今回の変更により Fable 5 の使用はトークン消費に基づくクレジット消費に統一される。Anthropic は「モデルの計算コストに応じた公平な利用管理」としている。API 経由のアクセスは従来通りトークン課金のまま変更なし。

Trump AI 大統領令の 8/1 実施期限が迫る ― フロンティアモデルの「分類済みベンチマーク」と政府事前アクセス枠組みが始動

Jul 8, 2026 セキュリティ

6 月 2 日にトランプ大統領が署名した大統領令 14409「AI イノベーションとセキュリティの促進」の主要条項が 8 月 1 日を実施期限として動き出している。中心となるのは NSA 主導の分類済みベンチマークの整備で、フロンティアモデルのサイバー能力を評価し「対象フロンティアモデル」に該当するかどうかを分類する。該当するモデルの開発者は、一般公開の最大 30 日前に政府機関への事前アクセスを任意で提供できる枠組みが設けられる。強制ライセンス・事前承認・許可制の導入は明示的に否定されており、あくまで自発的な協力スキームとして設計されている。

エンジニア的注目点: 「任意」とはいえ、政府との関係構築を優先する大手 AI 企業（OpenAI・Anthropic・xAI）にとっては事実上の対応圧力になる。分類済みベンチマークの基準が明らかになれば、何がフロンティアモデルとして扱われるかの境界線が見えてくる。国内でも AI 規制議論の参照点になりうる。

aion-labs が Aion-3.0 と Aion-3.0-Mini を公開 ― 推論特化のオープンモデル新世代

Jul 7, 2026 新モデル

aion-labs が Aion-3.0 および軽量版の Aion-3.0-Mini を 7 月 7 日に公開した。Aion シリーズは推論（reasoning）タスクへの特化を設計軸とするオープンモデル群で、数学・コード・科学分野での長ステップ推論に強みを持つ。Aion-3.0 はフルサイズで自前インフラへのデプロイを想定したスケール、Mini は API 利用・エッジデプロイ向けの小型版として提供されている。Hugging Face からダウンロード可能で、ライセンス条件は商用利用を一定範囲で許可している。独立ベンチマークでの位置付けは今週時点でまだ集積中。

Akeneo が Agentic Ziggy を発表 ― 製品クラウド内でデータモデリング・エンリッチメント・品質チェックを協調するエージェント層

Jul 8, 2026 エージェント

製品情報管理（PIM）ソフトウェアの Akeneo が Agentic Ziggy を発表。Akeneo Product Cloud 内に組み込まれたオーケストレーション層で、データモデリング・スキーママッピング・商品情報エンリッチメント・継続的品質チェックを担う専門エージェント群を調整する。エンドユーザーが操作する UI はそのままで、バックグラウンドでエージェントが商品データのメンテナンスを継続的に行う設計。EC・小売・製造業など商品カタログ管理を大量に行う業種を主な対象にしている。
